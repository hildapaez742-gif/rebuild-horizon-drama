# 角色一致性校验 Skill

## 定位

角色是一个完整的生命体，一致性覆盖五个维度：
**语言 × 情绪 × 行动 × 关系 × 成长**

五层缺一不可——台词对了但行动逻辑崩了，观众一样出戏。

---

## 工作流

### 阶段一：建立DNA档案（人物设计完成后执行一次）

读取 `references/dna-template.md`，根据已确认的人物设定，为每个核心角色生成完整的DNA档案。

档案包含：
- 身份层（表面/真实/知情者与时间线）
- 语言层（句式/口头禅/标志性表达/禁用表达）
- 情绪层（日常刻度/崩溃阈值/崩溃表现/情绪外露场景）
- 行动层（决策风格/面对陷阱/面对威胁/禁止行为）
- 关系层（对每个角色的态度差异）
- 成长轨迹（分阶段的状态变化）
- 校验触发词（出现时自动标记的关键条件）

**档案建立完成后：**
> "已为[角色名]建立DNA档案，包含X个校验触发条件。后续写作中将自动检测。"

### 阶段二：单集实时校验（每集生成后自动执行）

逐条扫描当集所有角色台词和动作描述：

```
扫描项目：
✓ 台词是否出现禁用表达
✓ 情绪刻度是否符合当前阶段设定
✓ 行动决策是否符合该角色逻辑
✓ 口头禅/句式习惯是否保持
✓ 对特定角色的态度是否符合关系层定义
✓ 是否出现校验触发词条件
```

输出：
- 一致性评分（0-100）
- 问题行标红 + 弹出说明
- 建议改写样本

### 阶段三：跨集漂移检测（每5集自动触发一次）

对比同一角色在当前集和5集前的表现，识别渐进式漂移：

```
检测项目：
- 台词平均句式长度变化
- 情绪外露度变化趋势
- 决策模式变化
- 与其他角色的语言风格相似度

判断：合理成长 vs 非预期漂移
```

**合理成长（不标记）：** 有剧情事件触发，变化符合成长轨迹
**非预期漂移（标记并提示）：** 无触发的风格突变，或与其他角色趋同

### 阶段四：全剧一致性报告（导出前触发）

读取 `references/consistency-report.md`，生成完整报告：
- 每个角色的一致性评分趋势图数据
- 漂移集数清单
- 角色间同质化程度（相似度超60%发出警告）
- 成长轨迹完整性检查

## 标准问题输出格式

**单集标记（在剧本编辑器内显示）：**

```
⚠️ 角色一致性警告

角色：[角色名]
位置：第X集·场景X·第X句
问题：[具体问题描述]
当前设定：[DNA档案中的对应规则]
检测到：[实际内容描述]

建议改写：
[具体改写样本，符合声纹标签]

[一键替换] [保留原文] [手动修改]
```

**跨集漂移提示：**

```
⚠️ 检测到角色漂移：[角色名]

第[N]集 → 第[N+5]集，以下维度出现非预期变化：
· 句式长度：平均从X字增加到X字
· 情绪外露度：从X成上升到X成（无剧情触发）
· 语言风格与[其他角色名]相似度达到XX%

建议：回顾第[N-N+5]集的[角色名]台词，
     按DNA档案中的[维度]标准进行调整。
```

## DNA档案动态更新规则

剧情推进中，发生影响角色设定的重大事件时，系统提示：

> "第X集[事件]发生，是否更新[角色名]的DNA档案？
> 建议调整：[具体调整项]"

**触发更新的事件类型：**
- 真实身份暴露
- 重大情感创伤或转折
- 核心关系发生质变
- 世界观崩塌（相信的东西被证明是假的）
- 主动选择放弃某个身份/能力/关系
