# 输出前自检清单

## 使用规则

**每集剧本输出前，内部执行本清单，不合格项自动修正后再输出。不要把检查过程输出给用户。**

---

## 大纲自检清单（第一阶段交付前执行）

### 结构完整性
- [ ] 全剧集数在建议范围内（28-35集）且有理由
- [ ] 情绪波浪各阶段都有规划（不能缺喘息期）
- [ ] 付费卡点位置已确定（第8-12集区间内）
- [ ] 至少5个主要反转已列出并标注集数
- [ ] 悬念类型规划已完成（无明显重复）

### 人物完整性
- [ ] 主角身份落差足够大（三级以上）
- [ ] 主角有具体的情感弱点
- [ ] 反派有第二层隐藏动机
- [ ] 牵绊人物已设计
- [ ] 所有核心角色有声纹锚点标签（3个/人）
- [ ] 核心人物总数≤4个

### 商业逻辑
- [ ] 免费钩子集（1-3集）的核心钩子已明确
- [ ] 试看断点设计在"最不甘心"位置，不在最高点
- [ ] 二次付费动机已设计（第20-25集区间）
- [ ] 金手指有激活条件+视觉表现+情感代价三要素

### 合规检查
- [ ] 无绝对禁止内容
- [ ] 高风险内容已处理或规避
- [ ] 结局有正向价值落点

---

## 单集自检清单（每集输出前执行）

### 节奏与结构
- [ ] 第一帧是否是全集最强视觉点？
- [ ] 上集悬念在本集前10秒是否有回应？
- [ ] 五步节拍是否都有体现（或有意识地调整了）？
- [ ] 本集悬念类型与上集不同？
- [ ] 情绪刻度是否符合本集在波浪图中的位置？

### 台词质量
- [ ] 有没有无法被画面还原的心理描写？→ 执行可视化转换
- [ ] 所有台词是否符合对应角色的声纹标签？
- [ ] 是否有超过15字没有断句的台词？→ 拆分
- [ ] 本集是否有至少一句嘴替金句？
- [ ] 台词是否带有情绪标记指令？

### AI配音适配
- [ ] 是否有高频多音字在歧义场景出现？→ 规避或注音
- [ ] 强情绪句是否以实词收尾？
- [ ] 超过12字的台词是否已强制断句？
- [ ] 是否使用了禁用词或慎用词？→ 替换
- [ ] 标点使用是否符合情绪系统规范？

### 剧本格式规范
- [ ] 场景标题行格式是否正确（内/外景 — 地点 — 时间）？
- [ ] 氛围行是否以△【氛围】开头，且是可视化感官细节？
- [ ] 动作行是否全部以△开头？
- [ ] 对白格式是否为「人物名：（情绪指令）内容」？
- [ ] 停顿标记是否使用[停顿]/[沉默]/——/……？
- [ ] 集尾悬念钩子是否有完整的标准块（类型/最后画面/转场）？
- [ ] OS独白是否超过每集2处限制？

### AI视频适配
- [ ] 场景描述是否有具体的可视化感官细节？
- [ ] 是否有禁止描写的动作类型？→ 用特效替换
- [ ] 场景数量是否合理（不超过3个场景/集）？

### 合规
- [ ] 台词中是否有可能触发审核的表达？→ 读取compliance.md处理

---

## 自检失败时的处理规则

发现不合格项时，**自动修正，不要询问用户是否需要修正**（除非涉及方向性的大改动）。

修正完成后直接输出修正版本。如果修正涉及较大改动，在输出后附一句说明：
> "这里有个地方做了调整：[一句话说明改了什么和原因]"

## 常见问题快速诊断

| 症状 | 可能原因 | 解决方向 |
|------|---------|---------|
| 台词读起来都像一个人说的 | 声纹锚点没有执行 | 重新对照每个角色的声纹标签改写 |
| 剧本看着很爽但感觉空洞 | 没有情感代价，金手指无限 | 给主角加入一个本集的情感损失 |
| 悬念感觉没力度 | 悬念类型重复，或卡点位置不对 | 换用不同类型悬念，断点移到更"不甘心"处 |
| 节奏拖沓 | 场景描述过多，或台词过长 | 删减描述，台词全部拆短 |
| 第一集感觉慢热 | 开头有铺垫 | 从最高冲突点切入，背景用台词带出 |
| AI配音听起来怪 | 断句错误或句末虚词 | 检查所有超过12字的句子，修正句末 |
