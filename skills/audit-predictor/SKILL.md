# 审核预判 Skill

## 定位

不是事后检查，是**全程伴随的风险雷达**。
不只指出问题，必须同时给出可落地的替换方案。

---

## 触发时机

| 时机 | 触发方式 | 扫描范围 |
|------|---------|---------|
| 大纲确认前 | 自动触发 | 全剧大纲 |
| 每集生成后 | 静默触发，有问题才提示 | 当前单集 |
| 导出前 | 用户确认后触发 | 全剧所有集数 |
| 用户手动触发 | 随时可调用 | 用户指定范围 |

---

## 五层扫描框架

读取 `references/risk-rules.md` 执行扫描，按以下顺序逐层检查：

```
第一层：硬红线（标🔴，必须修改，不可忽略）
  · 绝对禁止内容
  · 近期平台下架案例的相似表达

第二层：价值观风险（标🟠，强烈建议修改）
  · 阶层仇视倾向
  · 负面婚恋观无正向落点
  · 金钱崇拜无代价叙事
  · 反派行为被正向呈现

第三层：平台差异风险（标🟡，按目标平台判断）
  · 同一内容在不同平台风险等级不同
  · 需结合用户选定的目标平台评估

第四层：隐性风险（标🔵，提示关注）
  · 敏感词组合
  · 暴力细节密度
  · 结局无正向落点

第五层：结构性风险（标⚪，整体建议）
  · 主角获胜手段是否合法
  · 全剧价值观导向是否清晰
```

---

## 扫描执行逻辑

```
输入：[目标平台] + [内容类型：大纲/单集] + [具体内容]
        ↓
逐层扫描，命中规则时记录：
  · 位置（第X集 / 大纲第X部分 / 场景X第X句）
  · 风险层级（红/橙/黄/蓝/灰）
  · 原文引用（不超过20字）
  · 风险原因（一句话，15字内）
  · 替换方案（具体可执行，不是方向性建议）
        ↓
输出结构化风险报告
```

**输出格式（JSON结构，供工具解析）：**
```json
{
  "scan_time": "时间戳",
  "target_platform": "平台名",
  "total_risks": N,
  "risks": [
    {
      "level": "red/orange/yellow/blue/grey",
      "location": "第X集·场景X·第X句",
      "original": "原文内容",
      "reason": "风险原因",
      "suggestion": "具体替换方案",
      "platform_note": "平台差异说明（如有）"
    }
  ],
  "structural_notes": ["整体建议1", "整体建议2"]
}
```

## 平台差异处理

用户可选择目标平台，风险等级根据平台动态调整：

- **全平台最严**：取所有平台最高风险等级
- **抖音**：重点关注价值观导向，情绪强度相对宽松
- **快手**：重点关注低俗/软色情，家庭情感较友好
- **微信视频号**：全面审核，正向价值观要求最高
- **B站**：版权和政治敏感，创意表达相对宽松

---

## 替换方案生成原则

给出替换方案时必须遵守：
1. 保留原始的情感和戏剧效果，只换表达方式
2. 替换后的内容必须可以直接使用，不是方向性建议
3. 如果内容本身无法合规替换（涉及硬红线），明确告知并说明原因
4. 替换台词要符合对应角色的声纹标签

**替换后的处理方式：**
用户点击"一键替换"后，在对话区显示：
> "已将[位置]替换为建议内容，你看这样改是否合适？"
不静默修改原文，保持创作者控制感。
