# 热点雷达 Skill

## 定位

不是看榜单，是把热点数据转化为**选题决策依据**。
每次调用产出三样东西：赛道温度图 + 爆款结构解析 + 可执行选题建议。

---

## 工作流

### 第一步：并发搜索（4条查询同时发出）

```
查询1：短剧热榜 本周 抖音 [当前年月]
查询2：短剧付费榜 快手 最新
查询3：短剧 爆款 赛道 [当前年月]
查询4：AI短剧 AI漫剧 趋势 2026
```

搜索完成后读取 `references/analysis-framework.md`，按框架提炼数据。

### 第二步：结构化提炼

对搜索结果执行以下提炼，不转述原文，直接输出分析结论：

**赛道温度判断（每个赛道标注状态）：**
- 🔺 上升：近期新增作品多，完播率数据好
- → 平稳：有稳定受众，但无明显增长
- 🔻 饱和：同质化严重，新作突围难度高
- ⚠️ 高风险：近期有平台下架或政策收紧信号

**爆款结构解析（每部热门剧提取）：**
- 赛道 / 付费卡点位置 / 核心悬念类型 / 金手指类别
- 情绪主线 / 口碑关键词（来自评论区高频词）

**选题建议逻辑：**
对照 `references/analysis-framework.md` 中的赛道矩阵，输出当前最优选题方向，必须说明原因，不能只给结论。

### 第三步：输出报告

按标准输出格式（见下方）输出热点简报，结尾提供"用此方向创作"的一键跳转提示。

## 缓存与刷新规则

- 同一会话内，4小时内不重复搜索，直接使用上次结果
- 用户主动点击"刷新"或说"重新搜索"时强制刷新
- 每次刷新告知用户上次更新时间
